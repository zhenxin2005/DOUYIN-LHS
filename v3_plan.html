01

执行摘要

v3.0 将本工具从「静态知识库 + 定时发送」升级为「听觉 + 人格思维 + 自主决策 + 拟人输入」的人格化智能体,并从单一品类(龙虎山招财钱母)扩展为全品类可插拔架构。

四大核心升级:

🧬 立体人格八大人群 × 六生活维度组合,告别固定弹幕模板
🤖 LLM 驱动根据人格 + 主播语境实时思考,自然随机变化
🎧 双模听觉实时听主播 + 离线录播学产品,共享音频捕获底座
📚 多源采集文本/录音/读图归一到文本,统一提取品类知识包
设计哲学:分层降级,永不罢工。LLM 失败降级知识库,ASR 不可用降级定时,决策不回则静默——任何一层挂掉,系统自动回退到下一层可用模式。

03

核心设计原则:分层降级

整个 v3.0 的可靠性基石。任意环节异常,自动滑落到下一层可用模式,保证直播间永不「死寂」。

🧠 LLM 正常 → 生成人格化弹幕 ─ 失败/超时 → 📚 降级:知识库兜底话术
🎧 ASR 正常 → 实时听主播 ─ 不可用 → ⏰ 降级:定时发送(v2 模式)
🤔 决策层:该不该回? ─ 决策不回 → 🤫 静默(本就不该回)
✍️ 拟人输入 ─ 输入框丢失 → 📸 截图留证 + 重试(沿用 v2)

🧬 六维度驱动(立体人格)

八大人群只是基础层。真正的语气和回话方式,由一组「生活状态维度」组合决定——同样「小镇中老年」,用 800 元机还是 2000 元机、在江西还是在东北、丧偶还是已婚有娃,语气完全不同。这让 LLM 天然获得「随机变化」的素材。

维度字段影响
👥 人群crowd八大人群之一,基础画像
📱 手机device消费力侧面,影响打字速度/精致度
🚗 车vehicle身份信号,影响自我认同与话术
🛒 购物频次purchase_freq直播购物熟练度,影响提问方式
📍 省份region地域文化语气 + 对该品类的接受度基线
💍 婚姻marriage家庭责任视角,影响关注点(家宅/孩子/个人)

📐 人格 Schema

// rules.json — 维度驱动的人格定义 { "做生意的老张": { "crowd": "都市蓝领", // 八大人群 "device": "1500元安卓", // 手机价格档 "vehicle": "电动车", // 车(无车/10万国产/30万BBA) "purchase_freq": "中", // 高/中/低 "region": "河南", // 所在省份 "marriage": "已婚有娃", // 未婚/已婚无娃/已婚有娃/离异/丧偶 "tone": "直白务实、讲实惠", // 软画像(可手填,留空则LLM推导) "language_style": "短句少标点、爱问真有用吗", "response_tendency": 0.45, // 回应倾向 0~1 "typing_style": "slow", // 打字习惯 "messages": ["真有用吗", "便宜点不"] // 兜底话术 } }
软画像策略:tone / language_style 可手填(精细控制),也可留空让 LLM 据上面 6 个硬维度自行推导——手填优先,留空则推导。

产品知识采集(冷启动)

全品类 · 多源归一

📦 多源输入,文本归一

给客户的工具是全品类的。客户拿到手面对新品类,系统通过采集主播/客户提供的产品素材,最快学会产品知识。所有输入最终归一到文本,走统一 LLM 提取器,产出「品类知识包」。

📄
文本输入
.txt/.md/.docx 导入或粘贴
产品文档/话术稿
🎙️
录音输入
直播录音 → ASR 转文本
主播讲解,最鲜活
🖼️
图片输入
主图/详情页 → 视觉LLM读图
提取卖点+结构
⬇️ 全部归一为带来源标记的文本块
🧠 统一 LLM 提取器 → 品类知识包(JSON + 可读 MD)

🖼️ 读图方案(视觉 LLM 优先)

产品图(电商主图/详情页)排版复杂:大标题、卖点气泡、价格角标、对比表格。不要走纯 OCR——出来是乱序文字堆,分不清哪是标题哪是卖点。

方案能力适用
视觉 LLM(主) 看懂图意+排版+表格,直接结构化输出 产品主图、详情页、含图表的图
OCR(fallback) 只提文字,丢结构 纯文字图、无网络/省钱

🚀 客户冷启动标准流程

1️⃣ 提供 1~2 场直播录音 / 文档 / 产品图 → 2️⃣ 工具转写 + LLM 提取 → 3️⃣ 生成品类知识包 → 4️⃣ 客户 review 微调 → 5️⃣ 配置人格 → 6️⃣ 上线(听觉可选)
「最快」的诚实说明:录音是性价比最高(知识最鲜活),不是绝对最快。若客户有现成文档,直接导入分钟级上线;录音作为补充丰富。多源学习按客户手头素材选。
07

五大需求落地

1

人格定义(不只是编辑弹幕)

rules.json schema 升级:八大人群 × 六维度立体画像;messages 从「发送源」降级为「兜底话术 + 风格样本」;知识库关键信息独立成块供 LLM 引用。GUI「角色弹幕」标签页升级为「人格管理」表单。改造点:llm_engine.py
2

LLM 根据人格随机生成

新增 generate_message(persona, host_said, knowledge);Prompt = 人格小传 + 主播原话 + 知识库要点 + JSON 输出约束;温度 0.8~1.0 保证变化性;要求 LLM 自己决定回不回(respond: bool);失败/超时(>5s)降级到 messages 兜底。
3

听觉(能听不一定回)

Playwright 注入 JS,Web Audio API 抓 <video> 音频流 → 流式 ASR → 主播话语事件。能听 ≠ 要回:每条话语喂给人格大脑,由人格 response_tendency + 语境共同决定是否回应。ASR 不可用 → 降级 v2 定时模式。关键风险点:页面音频捕获需先 PoC。
4

思维(扮演真用户)

人格大脑统合需求 2/3:听 → 想 → 决策(回/不回)→ 回什么。节流:同角色冷却期内不重复回;全局速率上限防风控;决策不回则静默(本就不该回)。「真用户」的本质 = 有选择地回应,而非每条必回。
5

拟人输入(字/词分块,不过快)

替换当前 type(text, delay=30) 均匀输入:jieba 分词 → 词块序列 → 每块输入后随机停顿(50~200ms)→ 句中偶尔长停顿(0.5~1.5s 像在想)→ 速度服从人格 typing_style → 整体偏慢防风控。保留现有 React InputEvent 注入路径,只改节奏生成。
08

实施路线

M1 → M5 · 风险递增

核心策略:低风险先行,每阶段可独立交付价值。即使后期高风险环节失败,前期成果仍可用。

M1
人格化重构
rules.json 维度 schema + llm_engine 接 LLM 生成 + 知识库约束 + 失败降级
✅ 产出:LLM 驱动的人格弹幕(无听觉,定时触发)
M2
冷启动知识采集
knowledge_ingest 模块:文本/录音/读图 → 归一 → LLM 提取 → 品类知识包。给客户上手必需,且为 M4 验证音频捕获底座
✅ 产出:全品类冷启动能力 + 知识包
M3
拟人输入
字/词分块 + 随机节奏 + typing_style,替换均匀 type()
✅ 产出:真人感打字
M4
实时听觉高风险 · v0.x 翻车点
复用 M2 音频捕获底座,做流式 ASR + 主播话语事件。先 PoC 验证抖音页面音频可抓,不通过则搁置听觉
⚠️ 产出:实时听觉(可降级)
M5
决策融合 + 调优
ASR 事件 → 人格大脑 → 决策 → 拟人输入,全链路联调 + 节流防封 + 人格多样性调优
✅ 产出:v3.0 完整版
M1+M2+M3 即使 M4 再次失败,仍是 v2.x 的有效升级——这是对 v0.x 教训的核心回应:不再 all-in,分层可降级。

风险与对策

风险对策
高 ASR 拿不到页面音频(M4)PoC 先行,不通过则 v3 仅做 M1~M3,听觉搁置
中 LLM 跑题/不自然知识库约束 + 少样本(风格样本) + 长度/格式硬约束
中 LLM 成本/延迟决策层过滤(多数 respond=false 不调生成);缓存;超时降级
中 拟人输入仍被风控节流 + 抖动 + 频率上限;保留 v2 定时模式可切回
低 打包体积(本地模型)默认云 API;本地模型可选,不进打包

✓ 推荐

备选

备选

② LLM 方案(读图需求使其偏向视觉模型)

✓ 推荐

备选
DeepSeek(文本)+ 豆包/Qwen-VL(视觉) — 文本便宜,但两套模型。
备选
Ollama 本地 — 离线,但效果弱、体积大,不适合打包分发。

✓ 推荐
手配标杆 + 随机生成变体 — 核心 5 人群各手配 1 个标杆人格,再用随机维度组合铺出更多变体。

备选
纯随机 — 多样性强但质量参差。

✓ 推荐
